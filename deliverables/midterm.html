# Midterm Project

# CRUD API with API Integration

TIP

Due at 11:50PM on Friday Feb 21, 2025

Counts for 16% of your MAD9124 final grade.

# The Brief

Using Node.js, and Express, you will build the web service for a prototype of an application for tracking a resource of your choice. This resource will contain user data gathered from user input, as well as additional data obtained from an external API. This initial proof of concept (PoC) will have limited capabilities but should demonstrate the potential of a full-scale application.

# Core Requirements

  1. Using the Express.js framework, the API will cover all 6 CRUD operations on a resource.

  2. When created or updated, an additional request will be made to an external API, and data from the response will be saved along with the data from the original request body.

  3. The external API will be covered by a Redis caching layer.

  4. Be as creative as you want!

  • See this list of free apis for developers (opens new window) for inspiration
  1. Resource requirements:
  • Your resource needs to have:
key type description
id number unique, can be Date.now()
[user_input_1] any minimum of 2 fields from the user input
[user_input_2] any minimum of 2 fields from the user input
[api_field_1] any minimum of 1 field from the external API
  • Feel free to have more!

  1. Use at least 1 custom middleware function. This can be for validation, or even to do the API call ahead of time.

  2. Ensure that you write clean and readable code. Pay attention to:

  • No runtime errors
  • Logical grouping of related code
  • Semantically descriptive names for variables and functions
  • Well organized project folder structure
  • Do not commit your API key or node_modules to git!
  • Properly formatted package.json file
    • Correct project name
    • Your author details
  • Properly written package.json scripts (dev + start)
